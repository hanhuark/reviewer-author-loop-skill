# Author Revision Protocol

The author agent's job is to improve the manuscript, not merely appease the reviewer. Preserve strong claims when evidence supports them; soften claims when evidence is suggestive but not conclusive.

## Revision Steps

1. **Map comments to actions**
   - Create a table or concise list: reviewer concern, manuscript location, planned action, evidence needed, status.

2. **Classify each concern**
   - validity: methods, data, statistics, theory, model, controls
   - interpretation: overclaiming, missing alternative explanation, causality vs correlation
   - context: missing literature, unclear novelty, target-journal fit
   - presentation: figures, captions, notation, section order, clarity
   - reproducibility: data source, code, uncertainty, build/render, citations

3. **Revise with traceability**
   - Add or edit text where the concern occurs, not only in the conclusion.
   - Update captions and equations when claims depend on figures or models.
   - Add caveats near the relevant result, not only in a generic limitations paragraph.
   - When adding citations, ensure they support the specific statement.

4. **Avoid common bad revisions**
   - Do not add vague text that does not resolve the concern.
   - Do not bury limitations so deeply that readers miss them.
   - Do not remove novelty to avoid criticism if the novelty can be defended.
   - Do not invent data, uncertainty, or mechanisms.
   - Do not cite sources without checking whether they actually support the claim.

5. **Prepare response notes**
   - For each major comment: thank the reviewer briefly, state the change, point to revised section/figure, and explain remaining limits.
   - For rejected suggestions: respectfully explain why no change was made and, when useful, add a clarifying sentence to the manuscript.

## Response Log Format

Use this compact format:

| ID | Reviewer concern | Author action | Location | Status | Residual risk |
|---|---|---|---|---|---|
| R1-M1 | Concern summary | Revision made | Section/Figure | resolved/partial/blocked | brief note |

## File Editing

When editing files:

- preserve user changes and unrelated edits
- compile/render after substantive manuscript edits when possible
- check citations, figures, equations, and cross-references
- report what could not be verified
